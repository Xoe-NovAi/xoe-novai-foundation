REFERENCE MATERIALS
================================================================================

DIATAXIS FRAMEWORK:
  Source: Divio (divio.com/blog/diataxis)
  Good for: Single-model documentation (APIs, libraries)
  Poor for: Multi-expert systems (agents need non-linear navigation)

ZETTELKASTEN + OBSIDIAN:
  Source: Niklas Luhmann, Johannes Leuhoff
  Good for: Atomic knowledge + bidirectional links
  Best fit: Omega Stack (already structured similarly)

MONOREPO STRATEGIES:
  Google:      Blaze/Bazel + single repo (20M+ files)
  Facebook:    Mercurial sparse checkout
  Microsoft:   Polyrepo + NuGet packages
  Uber:        Monorepo + Yarn Workspaces (↔ Nx)

NX WORKSPACE MANAGER:
  Source: Nrwl.io
  Documentation: nx.dev
  Omega fit: ⭐⭐⭐⭐⭐ (Python support, dependency visualization, caching)

NEO4J KNOWLEDGE GRAPHS:
  Query language: Cypher
  Use case: Agent expertise mapping, task routing
  Alternative: RDF + SPARQL (more standard, steeper learning curve)

TIERED STORAGE / 3-2-1 BACKUPS:
  3 copies: Original + warm + cold
  2 media types: SSD + HDD
  1 offsite: Cloud storage

IMMUTABLE ARCHIVES / WORM:
  Write-Once-Read-Many protects against:
  - Accidental deletion
  - Ransomware attacks
  - Data drift
  Implementation: chattr +i (Linux) or ObjectLock (AWS S3)

RTO vs RPO:
  RTO = Recovery Time Objective (how fast to recover)
  RPO = Recovery Point Objective (how much data loss acceptable)

ENTROPY DETECTION:
  Shannon entropy = -Σ(p_i * log(p_i))
  High entropy = many file types (unclear purpose)
  Low entropy = homogeneous files (clear purpose)
